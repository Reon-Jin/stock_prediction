# A股智能分析系统 - 项目全流程概述

## 1. 项目简介

本项目是一套完整的A股量化分析系统，涵盖从数据采集到Web应用的全流程。系统核心目标是将原始市场数据转化为可训练、可预测、可解释的分析结果，并通过Web界面提供交互式分析能力。

## 2. 系统架构

系统采用四层架构设计：

### 2.1 数据层
- **职责**：从外部数据源采集原始数据并存储到MySQL
- **核心组件**：
  - `providers/`：数据源适配器（akshare、eastmoney等）
  - `jobs/`：数据同步任务
  - `warehouse/`：数据库模型和仓储访问

### 2.2 特征与样本层
- **职责**：将原始数据转换为模型可消费的特征和样本
- **核心组件**：
  - `features/`：特征工程模块
  - `labels/`：标签生成逻辑
  - `datasets/`：数据集切分和封装

### 2.3 模型层
- **职责**：模型训练、评估和推理
- **核心组件**：
  - `train/model.py`：模型定义
  - `train/run.py`：训练入口
  - `train/predict.py`：推理入口
  - `decision/`：决策和排序逻辑

### 2.4 应用层
- **职责**：提供用户交互界面
- **核心组件**：
  - `webapp/`：FastAPI后端
  - `frontend/`：React + Vite前端

## 3. 数据流程详解

### 3.1 数据采集阶段
1. **证券基础信息同步**
   - 同步股票列表、行业分类、交易状态等基础信息
   - 数据源：akshare

2. **行情数据同步**
   - 个股日线行情（开高低收、成交量、成交额）
   - 指数行情（沪深300、上证指数等）
   - 板块行情（行业板块、概念板块）
   - 资金流向数据

3. **基本面数据同步**
   - 财务快照（营收、利润、资产负债等）
   - 估值指标（PE、PB、PS等）

4. **文本数据同步**
   - 新闻数据（财经新闻、公司公告）
   - 公告数据（定期报告、临时公告）

### 3.2 特征工程阶段
1. **价格特征**
   - 移动平均线（MA5、MA10、MA20等）
   - 技术指标（MACD、RSI、KDJ等）
   - 价格动量、波动率特征

2. **量能特征**
   - 成交量变化率
   - 量价关系特征
   - 资金流向特征

3. **相对强弱特征**
   - 个股相对于行业的表现
   - 个股相对于市场的表现
   - 行业轮动特征

4. **基本面特征**
   - 财务指标变化率
   - 估值水平特征
   - 盈利能力特征

5. **市场环境特征**
   - 市场整体情绪
   - 行业热度
   - 市场波动率

6. **事件特征**
   - 新闻事件向量（256维）
   - 事件类型分类（业绩、政策、风险等）
   - 事件方向判断（利好、利空、中性）

7. **公司画像**
   - 公司基本面画像
   - 行业地位画像
   - 风险特征画像

8. **公司相似度**
   - 基于画像的公司相似度计算
   - 相似公司网络构建

### 3.3 样本构建阶段
1. **训练样本生成**
   - 将特征和标签组合成训练样本
   - 支持多种持有期标签（3日、5日、10日、20日、40日）
   - 生成多任务标签（胜率、收益、回撤、排序分数）

2. **数据切分**
   - 时间序列切分（非随机切分）
   - 训练集：2023-01-01 ~ 2024-12-31
   - 验证集：2025-03-01 ~ 2025-06-30
   - 测试集：2025-09-01 ~ 2025-12-31
   - 间隔期：60天

3. **数据导出**
   - 训练集：`train.parquet`
   - 验证集：`valid.parquet`
   - 测试集：`test.parquet`
   - 事件数据：`*_events.parquet`

### 3.4 模型训练阶段
1. **模型架构**
   - 多尺度时序编码器（TemporalConv + BiGRU + Self-Attention）
   - 深层MLP编码器（带残差门控）
   - 公司Embedding层
   - 相似公司加权池化
   - 上下文门控调制
   - 多任务输出头

2. **训练策略**
   - 自动混合精度训练（AMP）
   - 早停机制
   - 学习率调度
   - 正负样本加权
   - 多周期损失加权

3. **训练产物**
   - 最佳模型：`best_model.pt`
   - 训练历史：`history.csv`、`history.json`
   - 训练曲线：`training_curves.png`
   - 业务指标：`business_curves.png`
   - 测试指标：`test_metrics.json`

### 3.5 模型推理阶段
1. **推理输入**
   - 今日预测样本（`today_infer.parquet`）
   - 或历史测试样本

2. **推理输出**
   - 胜率概率：`p_win_prob_*`
   - 预期收益：`ret_mu_pred_*`
   - 风险回撤：`risk_dd_pred_*`
   - 决策分数：`decision_score_*`
   - 排序分数：`rank_score_pred`
   - 信号分数：`signal_score`

3. **结果输出**
   - Parquet格式预测结果
   - CSV格式预测结果

## 5. 部署和使用

### 5.1 环境要求
- Python 3.10+
- MySQL 8.x
- Node.js 18+
- CUDA（可选，用于GPU训练）

### 5.2 部署步骤
1. **环境准备**
   ```bash
   # 创建Python环境
   conda create -n stock_prediction python=3.12
   conda activate stock_prediction
   pip install -r requirements.txt
   
   # 安装前端依赖
   cd frontend && npm install
   ```

2. **配置修改**
   - 修改`configs/config.yaml`中的数据库连接
   - 配置数据源参数
   - 配置LLM API密钥

3. **数据库初始化**
   ```bash
   python -m warehouse.schema_init
   ```

4. **数据同步和训练**
   ```bash
   # 全量数据同步
   python scripts/run_full_pipeline.py --start 2023-01-01 --end 2025-12-31
   
   # 模型训练
   python -m train.run
   ```

5. **Web应用启动**
   ```bash
   # 启动后端
   uvicorn webapp.main:app --host 127.0.0.1 --port 8000
   
   # 启动前端
   cd frontend && npm run dev
   ```

### 5.3 日常使用
1. **数据更新**
   ```bash
   # 同步最新数据
   python scripts/run_full_pipeline.py --today
   ```

2. **模型推理**
   ```bash
   # 生成今日预测
   python -m train.predict --checkpoint-path train/artifacts/best/best_model.pt --input-path data/today_infer.parquet
   ```

3. **Web界面使用**
   - 访问http://127.0.0.1:5173
   - 注册/登录账户
   - 使用单股分析或全市场推荐功能
